Uma Mensagem do Professor Carvalho

Olá, treinador!

Sou o Professor Carvalho, pesquisador Pokémon da região de Kanto.

Nos últimos anos, meu laboratório tem estudado novas formas de registrar informações sobre Pokémon utilizando sistemas computacionais. Entretanto, nossos computadores ainda não são capazes de reproduzir adequadamente as batalhas que acontecem entre treinadores.

Por isso, preciso da sua ajuda.

Sua missão será desenvolver uma aplicação web capaz de simular visualmente uma batalha Pokémon inspirada nos jogos clássicos da franquia, utilizando tecnologias modernas de desenvolvimento web.

Durante esta pesquisa, você precisará consultar a Pokédex digital (PokeAPI), coletar informações dos Pokémon e exibi-las em uma interface semelhante às batalhas vistas nos jogos Pokémon Fire Red, Leaf Green ou Emerald.

Boa sorte, treinador. Estou ansioso para ver os resultados da sua pesquisa!

⸻

Objetivo

Desenvolver uma página web utilizando:

* HTML
* CSS
* JavaScript
* PokeAPI

O objetivo principal é reproduzir visualmente uma batalha Pokémon semelhante às dos jogos clássicos.

A batalha não precisa estar funcional.

O foco principal desta atividade é:

* Consumir dados da PokeAPI;
* Trabalhar com HTML, CSS e JavaScript;
* Manipular o DOM;
* Construir uma interface inspirada em um jogo real.

⸻

Procure reproduzir o máximo possível dos elementos presentes na imagem:

* Cenário;

Estrutura Obrigatória do Projeto

Todos os arquivos, exceto o index.html, deverão estar dentro da pasta assets.

Utilize obrigatoriamente a seguinte estrutura:

projeto/
│
├── index.html
│
└── assets/
    │
    ├── css/
    │   └── style.css
    │
    ├── js/
    │   └── script.js
    │
    ├── img/
    │   ├── referencia-batalha.png
    │   ├── pokemon-logo.png
    │   └── demais imagens
    │
    └── audio/
        └── battle-theme.mp3

Requisitos Obrigatórios

1. Consumir a PokeAPI

Sua aplicação deverá obter informações reais dos Pokémon utilizando a PokeAPI.

Link:

https://pokeapi.co/

Você pode utilizar informações como:

* Nome;
* Sprites;
* Tipos;
* HP;
* Altura;
* Peso;
* Habilidades;
* Movimentos.

⸻

2. Construir a Interface de Batalha

A tela deverá possuir, no mínimo:

* Pokémon adversário;
* Pokémon do jogador;
* Nome dos Pokémon;
* Nível dos Pokémon;
* Barra de HP;
* Caixa de diálogo;
* Menu de golpes;
* Informações básicas obtidas da API.

⸻

3. Utilizar HTML, CSS e JavaScript Puro

Não é permitido utilizar:

* Bootstrap;
* Tailwind;
* Material UI;
* React;
* Vue;
* Angular;
* Engines de jogos;
* Templates prontos.

O objetivo é praticar HTML, CSS e JavaScript puro.

⸻

4. Organização e Aparência

O projeto deverá apresentar:

* Estrutura organizada;
* Código identado;
* CSS organizado;
* Interface semelhante à referência;
* Boa experiência visual.

⸻

Música de Batalha

Toda batalha Pokémon precisa de uma boa trilha sonora.

Adicione uma música de fundo inspirada nas batalhas clássicas da franquia Pokémon.

Você pode utilizar:

* Temas de batalha Pokémon;
* Música 8-bits;
* Música retrô;
* Música de RPG.
